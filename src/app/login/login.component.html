Welcome to IMS Project

Overview

The aim of the project is to create an application which will automate the management of internships for the students, starting from the candidacy up to the validation.

1 The roles

We defined four primary roles which are the actors handled by the system.

1.1 The student

The student can search/apply to an offer, generate an internship agreement and validate a completed internship.

1.2 The class coordinator​ (CC)​ : this role represents a manager of a classroom

A class coordinator can control the applications and the process of internship agreements elaboration he handles.

1.3 The partner ​: this role is linked to an enterprise, a laboratory or a school making internships available.

A partner can manage the application of a student to his offers and the internship agreement.

1.4 The security and defence officer (SDO)

A SDO can control applications of students when they apply to offers in need of security protocols.
We defined two other roles for a well functioning system :

1.5 The administrator

An administrator has full control over the application. He will be able to look at/edit/delete all the offers, applications, internship agreements and archived internships.

1.6 The anonymous person

An anonymous person can not do anything on the system. An anonymous can only log in the system
